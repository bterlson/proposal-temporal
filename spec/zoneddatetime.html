ZonedDateTime Objects

A `ZonedDateTime` object is an immutable object referencing a point in time. The referenced point in time is specific to a given time zone and is measured with nanoseconds precision.

ZonedDateTime Constructor

The ZonedDateTime constructor is the %ZonedDateTime% intrinsic object. When called as a constructor, it creates a new ZonedDateTime object. *instant* is an Instant object and *timeZone* is a String corresponding to an IANA time zone.

The ZonedDateTime constructor is a single function.

The ZonedDateTime constructor is subclassable.

The length property of the ZonedDateTime construtor function is 2.

Properties of the ZonedDateTime Constructor

The value of the [[Prototype]] internal slot of the ZonedDateTime constructor is the intrinsic object %FunctionPrototype%.

The ZonedDateTime constructor has the following properties:

ZonedDateTime(*instant*, *timeZone*)

This description only applies if the ZonedDateTime constructor is called with at least 2 arguments.

When the ZonedDateTime constructor function is called, the following steps are taken:

1. Let numberOfArgs be the number of arguments passed to this function call. 1. Assert: numberOfArgs == 2. 1. If NewTarget is not undefined, then 1. Let ms be instant.milliseconds. 1. Let ns be instant.nanoseconds. 1. Let tz be timeZone. 1. Let O be ? OrdinaryCreateFromConstructor(NewTarget, "%ZonedInstantPrototype%", <<[[MillisecondValue, NanosecondValue,TimeZoneValue]]>>). 1. Set O.[[MillisecondValue]] to ms. 1. Set O.[[NanosecondValue]] to ns. 1. Set O.[[TimeZoneValue]] to tz. 1. Return O.

ZonedDateTime Prototype

The initial value of ZonedDateTime.prototype is the intrinsic object %ZonedDateTime%. This property has the attributes {[[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false}.

ZonedDateTime.prototype.year

`zonedDateTime.year` is a getter that returns the year Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.month

`zonedDateTime.month` is a getter that returns the month of the year as a Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.day

`zonedDateTime.day` is a getter that returns the day of the month as a Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.hour

`zonedDateTime.hour` is a getter that returns the hour of the day as a Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.second

`zonedDateTime.second` is a getter that returns the second of the minute as a Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.millisecond

`zonedDateTime.millisecond` is a getter that returns the millisecond part of a second as a Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.microsecond

`zonedDateTime.microsecond` is a getter that returns the microsecond part of the millisecond as a Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.nanosecond

`zonedDateTime.nanosecond` is a getter that returns the nanosecond part of the microsecond as a Number integer according to the proleptic gregorian calendar and a 24h day represented by the ZonedInstant in its timezone.

ZonedDateTime.prototype.instant

`zonedDateTime.instant` is a getter that returns the `Instant` of this `zonedDateTime` unrelated to a timezone.

ZonedDateTime.prototype.offsetSeconds

`zonedDateTime.offsetSeconds` is a getter that returns the offset of this `zonedDateTime` and its timezone from UTC expressed as the Number of seconds from UTC.

ZonedDateTime.prototype.offsetString

`zonedDateTime.offsetString` is a getter that returns the offset of this `zonedDateTime` and its timezone from UTC expressed as a String with the format `[+-]hh:ss`

ZonedDateTime.prototype.ianaZone

`zonedDateTime.ianaZone` is a getter that returns the geographic IANA timezone if it was specifiedwhen creting the `ZonedDateTime`. or `undefined` otherwise.

ZonedDateTime.prototype.timeZone

`zonedDateTime.ianaZone` is a getter that returns the geographic IANA timezone if it was specifiedwhen creting the `ZonedDateTime` otherwise it returns the offset of this `zonedDateTime` and its timezone from UTC expressed as a String with the format `[+-]hh:ss`

ZonedDateTime.prototype.plus({ ? _years_, ? _months_, ? _days_, ? _hours_, ? _minutes_, ? _seconds_, ? _milliseconds_, ? _nanoseconds_ })

When called with a ZonedDateTime object as the _this_ context, the result will be a new ZonedDateTime object.

The value of the new ZonedDateTime object is the value of the original ZonedDateTime with the specified date parts added. Units will be added in order of size, ascending.

Calling this function with a _this_ context that is not an instance of ZonedDateTime is illegal.

ZonedDateTime.prototype.toCivilDate()

The abstract operation toCivilDate returns a new CivilDate Object with the year, month, and day values calculated from the milliseconds and nanoseconds since Epoch

ZonedDateTime.prototype.toString()

When called with a ZonedDateTime object as a _this_ context it will return an ISO 8601 String value representing the value of the ZonedInstant object.

The format of the String will always be an ISODateTime of the following form:
