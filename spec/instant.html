Instant Objects

An `Instant` is an immutable object referencing a point in time with nanoseconds precision.

new Instant(*nanoseconds*)

When called as a constructor creates a new Instant object. *nanoseconds* is a BigInt of the nanoseconds since epoch. This value is stored in the [[value]] slot.

Instant.prototype.seconds

`instant.seconds` is a getter that returns the seconds Number integer since epoch of the point in time represented by the instant object such that `instant.seconds === Math.floor([[value]] / 1e9)` is *true*.

Instant.prototype.milliseconds

`instant.milliseconds` is a getter that returns the milliseconds Number integer since epoch of the point in time represented by the instant object such that `instant.milliseconds === Math.floor([[value]] / 1e6)` is *true*.

Instant.prototype.microseconds

`instant.mircoseconds` is a getter that returns the microseconds BigInt since epoch of the point in time represented by the instant object such that `instant.microseconds === Math.floor([[value]] / 1e3)` is *true*.

Instant.prototype.milliseconds

`instant.mircoseconds` is a getter that returns the nanoseconds BigInt since epoch of the point in time represented by the instant object such that `instant.nanoseconds === [[value]]` is *true*.

Instant.prototype.withZone(_timezone_)

When called with an Instant object as a _this_ context it will return a ZonedDateTime object that specifies an instant in time in the given time zone.

_timezone_ is a string specifying a time zone. Legal values are:

  • a IANA geographic time zone
  • an offset string in the format `[+-]hh:mm`
  • the string *"UTC"* indicates that the time zone should be Coordinated Universal Time.
  • the string *"SYSTEM"* indicates that the time zone should be local time as provided by the system.

Calling this function with a _this_ context that is not an instance of Instant is illegal.

Instant.prototype.toString()

When called with an Instant object as a _this_ context it will return an ISO 8601 String value representing the value of the Instant object.

The format of the String will always be an IsoDateTime of the following form:

Sign: `-` Year: Sign? Digit Digit Digit Digit Month: `01` through `12` Day: `01` through `31` Hour: `00` through `24` Minute: `00` through `59` Second: `00` through `59` Nanoseconds: `000000000` through `999999999` IsoDateTime: Year Month Day `T` Hour Minute Second `.` Nanoseconds `Z`

Calling this function with a _this_ context that is not an instance of Instant is illegal.

Instant.prototype.toJSON()

When called with an Instant object as a _this_ context it will return an ISO 8601 String value representing the value of the Instant object equivalent to `this.toString()`.

Instant.fromSeconds(_seconds_)

This function will create an Instant object that represents the instant in time _seconds_ since epoch. Lower precision will be zeroed.

_seconds_ is the Number of seconds since epoch.

Instant.fromMilliseconds(_milliseconds_)

This function will create an Instant object that represents the instant in time _milliseconds_ since epoch. Lower precision will be zeroed.

_milliseconds_ is the Number of milliseconds since epoch.

Instant.fromMicroseconds(_microseconds_)

This function will create an Instant object that represents the instant in time _microseconds_ since epoch. Lower precision will be zeroed.

_microseconds_ is the BigInt of microseconds since epoch.

Instant.fromNanoseconds(_nanoseconds_)

This function will create an Instant object that represents the instant in time _nanoseconds_ since epoch.

_nanoeconds_ is the BigInt of nanoeconds since epoch.

Instant.fromUTC(_year_, _month_, _day_, _hour_, _minute_, _second_, _millisecond_, _microsecond_, _nanosecond_)

This function will create an Instant object that represents the instant in time specified as a *UTC* date since epoch.

_year_ is the year as a Number according to the proleptic gregorian calendar.

_month_ is the month as a Number according to the proleptic gregorian calendar. The value is between `1..12`.

_day_ is the day of the month as a Number according to the proleptic gregorian calendar. The value is between `1..31` depending on the month.

_hour_ is the hour of the day as a Number based on a 24h midnight based day. The value is between `0..23`.

_minute_ is the minute of the hour as a Number with a value between `0..59`.

_second_ is the second of the minute as a Number with a value between `0..60`. The value `60` is used for leap-seconds.

_millisecond_ is the millisecond part as a Number with a value between `0..999`.

_microsecond_ is the microsecond part as a Number with a value between `0..999`.

_nanosecond_ is the nanosecond part as a Number with a value between `0..999`.

Instant.fromString(_string_)

This function will create an Instant object from a String. The _string_ has to strictly conform to the format produced by . Any _string_ not strictly conforming to that format will throw an Error.
