Duration Objects

Duration objects describe the difference between two like Date, Time, or DateTime objects. They are only created via the .minus() methods of these objects.

  • The .minus() operation on Date objects, creates a Duration that only has years, months and days properties.
  • The .minus() operation on Time objects, creates a Duration that only has hours, minutes, seconds, milliseconds, microseconds and nanoseconds properties.
  • The .minus() operation on Instant objects, creates a Duration like that of Time objects.
  • The .minus() operation on DateTime objects , creates a Duration that has all member properties, except if passed an option indicating it should be created as a Time object or Date object.

When created, the smaller units are combined into full larger units (i.e. 1000 milliseconds are recorded as 1 seconds) up to the largest available unit. The result of this is that the largest available unit may have a larger value in some circumstances. Durations created from Time objects may have an hours property larger than 24, which would be invalid for a Duration created from a DateTime object.

When a Duration is applied to an object via the .plus() method, only the relevant fields are taken into consideration. Such that a Date objects receiving a DateTime duration will entirely ignore the values of the hours and below fields. All properties of Durations are positive integers. When coerced from a plain object that looks like a Duration the following rules are applied:

  1. All fields whose names match those of a Duration are extracted.
  2. Each value is coerced:
    • If the field is numeric it is rounded to the nearest integer.
    • If the field is non-numeric it is set to undefined.
  3. A Duration is created with the available values.
  4. If one of years, months or days is numeric, the others are set to 0.
  5. If one of hours, minutes, seconds, milliseconds, microseconds or nanoseconds is numeric, the others are set to 0.
  6. The Duration is normalized through adding/subtracting values of adjacent units.
  7. If the resulting Duration has a negative value that value is converted to its absolute value.

Duration objects have the following (optional) properties:

  • years
  • months
  • days
  • hours
  • minutes
  • seconds
  • milliseconds
  • microseconds
  • nanoseconds
